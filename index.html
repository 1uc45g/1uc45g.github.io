  • Bien assimiler des notions importantes comme le DNS.

Projet : Découverte du web hacking et de la sécurisation du web

Objectif du projet : Découverte du web hacking en exploitant les failles sur une machine volontairement vulnérable (DVWA : Damn Vulnerable Web Application) puis en produisant un rapport écrit ainsi qu'oral accompagnés de démonstrations sur notre exploitation.

DVWA est une application web volontairement rendue vulnérable mise à disposition des utilisateurs afin qu'ils puissent s'entraîner aux tests de sécurité dans un environnement contrôlé.
Projet effectué en duo, en 2023-2024
  • Exploitation de 16 failles différentes, à plusieurs niveaux de difficulté ("low", "medium" et "high").
  • Exemple avec la faille "weak session IDs": À chaque fois que l’on clique sur le bouton présenté dans la page associée à cette faille, on génère une valeur permettant d'identifier une session. Pour l'exploitation de cette faille, on cherche à identifier un modèle pour prédire les valeurs générées, pour s'emparer de sessions :
  • Analyse du code de pages web pour apprendre à les sécuriser lors de leur conception.
  • Identification d'une solution associée à cette faille: générer un identifiant en concaténant un nombre pseudo-aléatoire avec l'heure et une chaîne de caractères, puis en hachant le tout, ici avec l'algorithme "sha1".
  • Ce projet m’a permis d’apprendre à analyser un site web, afin d'en améliorer la conception, ainsi qu'à rendre compte à l'écrit et à l'oral, accompagnant une démonstration d'explications.

Project: Discovering web hacking

Project objective: Discovery of web hacking in exploiting the vulnerabilities on a deliberately vulnerable machine (DVWA: Damn Vulnerable Web Application) then by producing a written and oral report on our exploitation.

DVWA is a voluntarily made vulnerable web application made available to users so that they can practice security testing in a controlled environment.
Project carried out in duo, in 2023-2024
  • Exploitation of 16 different vulnerabilties, at several difficulty levels (low, medium and high).
  • Example with the vulnerability "weak session IDs": Each time we click on the button presented in the page associated with this vulnerability, we generate a value to identify a session. For the exploitation of this vulnerability, we seek to identify a model to predict the values generated, to seize sessions:
  • Analysis of the code of web pages to learn how to secure them in their design
  • Identification of a solution associated with this vulnerability: generate an identifier by concatenating a pseudo-numberrandom with time and string, then hashing everything, here with the "sha1" algorithm.
  • This project allowed me to learn how to analyze a website, in order to improve the design, as well as to report in writing and orally, accompanied by a demonstration of explanations.

Projet : Caractérisation des réseaux de téléphonie mobile
