Projet : concevoir le réseau informatique d'une petite entreprise

Objectif du projet : Concevoir entièrement, sur Cisco Packet Tracer, le réseau local d'une petite entreprise pour répondre à ses besoins en commutation, routage, services réseau de base et sécurité.

  • Concevoir une architecture d’adressage en IPv4 pour les machines du réseau, en VLSM, à l’aide de tableaux.
  • Répartir les différents équipements réseau dans le LAN de l'entreprise et dans la DMZ.
  • Configurer des commutateurs en mode spanning-tree avec les bonnes commandes.
  • Entrer les enregistrements nécessaires au bon fonctionnement d'un serveur DNS privé et de plusieurs serveurs DNS publics.
  • Configurer plusieurs fonctionnalités dans un routeur telles que le routage inter-VLAN, la traduction d’adresses, un serveur DHCP.

Ce projet m’a permis d’apprendre à dépanner des fonctionnalités comme l’attribution d’une configuration via le DHCP au sein d’un LAN en utilisant une approche rigoureuse pour la résolution des dysfonctionnements, et à bien réfléchir à la conception avant de débuter la configuration des équipements du réseau. J'ai ainsi pu améliorer mes compétences en matière d'administration des réseaux.

Principales difficultés rencontrées :
  • Se concentrer pendant de longues périodes afin de produire un résultat satisfaisant.
  • Bien comprendre des notions importantes comme le DNS.

Projet : traiter les données de fichiers de log

Objectif du projet : créer un programme en python, générant une page web de type «dashboard» (tableau de bord) qui affiche des informations textuelles tirées d'un fichier de log ainsi que des graphiques.

Les fichiers utilisés dans le cadre de ce projet correspondent à des fichiers de type calendrier (.ics).
  • Concevoir le programme pour qu'il soit adapté aux besoins de l'utilisateur en fonction de la structure du contenu du fichier de log à analyser.
  • Utiliser les expressions régulières pour identifier les différents évènements.
  • Utiliser les f-string pour formater des tableaux et les informations sur la page web.
  • Ce projet m’a permis d’apprendre à déceler des problèmes dans un programme et à les résoudre, mais aussi à documenter du code pour d’autres utilisateurs. Ce projet m'a donc permis d'améliorer mes compétences en programmation, notamment en documentant le travail réalisé, en choisissant les outils de développement adaptés. On peut également établir un lien avec les compétences de surveillance d'un système, puisque j'ai créé un outil automatisant un tâche qui consiste à analyser un fichier de log, permettant par la suite l'exploitation des informations qu'il contient.
  • L’une des principales difficultés rencontrées : se concentrer pendant de longues périodes afin de produire un résultat satisfaisant.
  • Points d’amélioration : sur le mode de pensée algorithmique et la manière de documenter le code (se concentrer sur les lignes les plus complexes, donc difficilement compréhensibles par l’utilisateur)
